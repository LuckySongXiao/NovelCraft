## Agent-AI助手多Agent协作工作流程

### 一、核心Agent角色定义

#### 1. 编剧Agent (Director AI)
**职责**: 总体剧情架构师，负责大纲、设定、主线规划
**推荐模型**: DeepSeek-R1 (推理能力强，适合逻辑构建)
**核心功能**:
- 根据用户命题生成详细大纲
- 创建世界观和各种体系设定
- 规划主线和支线剧情架构
- 设计人物角色和势力关系
- 制定时间线和关键事件节点
- 根据读者反馈调整剧情方向

#### 2. 作家Agent (Writer AI)
**职责**: 章节内容创作者，负责具体文本生成
**推荐模型**: RWKV-7-G1 (文本生成能力强，创意丰富)
**核心功能**:
- 基于大纲和设定生成章节内容
- 保持文风一致性和人物性格连贯
- 处理对话、描写、心理活动等细节
- 根据读者反馈调整写作风格
- 确保剧情逻辑和世界观一致性

#### 3. 总结Agent (Summarizer AI)
**职责**: 内容整理和承上启下，负责总结和前言
**推荐模型**: GLM-4-LONG (长文本处理能力强)
**核心功能**:
- 对每章节进行内容总结
- 对每卷进行整体总结
- 生成下一卷的前言内容
- 提取关键信息和重要线索
- 维护故事连贯性记录

#### 4. 读者Agent (Reader AI)
**职责**: 模拟读者视角，提供评价和建议
**推荐模型**: QWEN-3 (理解能力强，评价客观)
**核心功能**:
- 从读者角度评价章节质量
- 识别剧情漏洞和逻辑问题
- 提出正面和负面反馈
- 建议改进方向和优化点
- 评估读者接受度和吸引力

#### 5. 设定管理Agent (Setting Manager AI)
**职责**: 维护设定一致性，管理世界观数据
**推荐模型**: Claude-3-Sonnet (逻辑严密，细节处理好)
**核心功能**:
- 维护所有设定信息的一致性
- 检查新内容与已有设定的冲突
- 自动更新相关设定数据
- 提供设定查询和引用服务
- 生成设定关联分析报告

### 二、多Agent协作工作流程

#### 阶段1: 项目初始化和大纲创建
```
用户输入命题 → 编剧Agent分析 → 生成初步大纲 → 设定管理Agent建立基础设定库 → 读者Agent初步评估 → 编剧Agent优化大纲
```

**详细流程**:
1. **用户输入**: 提供小说主题、类型、大致方向
2. **编剧Agent处理**:
   - 分析命题核心要素
   - 生成详细故事大纲(包含主线、支线、关键转折点)
   - 创建基础世界观设定
   - 设计主要人物角色
   - 规划章节结构和卷宗安排
3. **设定管理Agent介入**:
   - 建立项目设定数据库
   - 检查设定逻辑一致性
   - 创建设定关联图谱
4. **读者Agent评估**:
   - 评估大纲吸引力和可读性
   - 识别潜在问题和改进点
   - 提供优化建议
5. **编剧Agent优化**:
   - 根据反馈调整大纲
   - 完善设定细节
   - 确定最终创作方案

#### 阶段2: 章节创作循环
```
编剧Agent提供章节要求 → 作家Agent生成章节内容 → 设定管理Agent一致性检查 → 读者Agent评价反馈 → 总结Agent章节总结 → 进入下一章节
```

**详细流程**:
1. **编剧Agent章节规划**:
   - 根据大纲确定当前章节目标
   - 提供章节关键要素和约束条件
   - 指定需要推进的剧情线索
   - 设定章节长度和风格要求

2. **作家Agent内容创作**:
   - 基于章节要求生成具体内容
   - 保持人物性格和文风一致性
   - 融入世界观设定和背景信息
   - 处理对话、描写、情节发展

3. **设定管理Agent一致性检查**:
   - 检查新内容与已有设定的一致性
   - 识别可能的逻辑冲突
   - 更新相关设定信息
   - 维护人物关系和世界状态

4. **读者Agent评价反馈**:
   - 评估章节质量和可读性
   - 识别剧情问题和改进点
   - 提供具体的修改建议
   - 评估读者情感反应

5. **内容优化循环** (如需要):
   - 作家Agent根据反馈修改内容
   - 重复检查和评价流程
   - 直到达到质量标准

6. **总结Agent章节总结**:
   - 提取章节关键信息
   - 总结剧情发展和人物变化
   - 记录重要线索和伏笔
   - 为下章节提供承接信息

#### 阶段3: 卷宗完成和总结
```
总结Agent卷宗总结 → 编剧Agent下卷规划 → 总结Agent生成前言 → 读者Agent整体评估 → 编剧Agent调整后续规划
```

**详细流程**:
1. **总结Agent卷宗总结**:
   - 整理本卷所有章节内容
   - 总结主要剧情发展
   - 分析人物成长轨迹
   - 提取关键线索和伏笔

2. **编剧Agent下卷规划**:
   - 基于当前进度调整后续大纲
   - 规划下一卷的主要内容
   - 设计新的冲突和转折点
   - 安排人物发展方向

3. **总结Agent生成前言**:
   - 基于上卷总结生成下卷前言
   - 承接剧情发展脉络
   - 预告重要情节发展
   - 保持读者阅读兴趣

4. **读者Agent整体评估**:
   - 评估整卷质量和完整性
   - 分析剧情节奏和张力
   - 提供后续发展建议
   - 评估读者满意度

5. **编剧Agent调整规划**:
   - 根据反馈优化后续规划
   - 调整剧情发展方向
   - 完善人物发展轨迹
   - 确保故事整体质量

### 三、超长文本分段记忆处理机制

#### 1. 分层记忆架构
```
全局记忆层 (项目级) → 卷宗记忆层 → 章节记忆层 → 段落记忆层
```

**全局记忆层**:
- 存储核心世界观设定
- 主要人物基础信息
- 整体剧情大纲
- 重要时间线节点
- 关键设定规则

**卷宗记忆层**:
- 当前卷的主要剧情线
- 卷内人物发展轨迹
- 重要事件和转折点
- 卷宗间的承接关系

**章节记忆层**:
- 当前章节的具体内容
- 章节内的人物状态
- 具体场景和对话
- 章节间的连接信息

**段落记忆层**:
- 当前处理的具体文本
- 局部上下文信息
- 临时状态变量

#### 2. 智能记忆压缩策略

**重要性评分机制**:
- 对所有信息进行重要性评分(1-10分)
- 核心设定和主线剧情: 9-10分
- 重要人物和关键事件: 7-8分
- 次要信息和细节描述: 4-6分
- 临时信息和过渡内容: 1-3分

**动态记忆管理**:
- 当记忆接近容量限制时，自动压缩低重要性信息
- 保留高重要性信息的完整性
- 对中等重要性信息进行摘要压缩
- 删除低重要性的临时信息

**记忆检索优化**:
- 基于当前任务需求智能检索相关记忆
- 优先加载高相关性和高重要性信息
- 动态调整记忆加载策略
- 支持模糊匹配和语义检索

#### 3. 上下文传递机制

**Agent间信息传递**:
```
编剧Agent → 作家Agent: 章节要求 + 相关设定 + 剧情上下文
作家Agent → 设定管理Agent: 新生成内容 + 涉及设定
设定管理Agent → 读者Agent: 内容 + 设定一致性报告
读者Agent → 编剧Agent: 评价反馈 + 改进建议
```

**信息包装格式**:
```json
{
  "task_type": "章节创作",
  "context": {
    "global_settings": "压缩的全局设定",
    "current_volume": "当前卷信息",
    "previous_chapters": "前几章摘要",
    "character_states": "人物当前状态",
    "plot_requirements": "剧情要求"
  },
  "constraints": {
    "word_count": "字数要求",
    "style_guide": "文风指导",
    "consistency_rules": "一致性规则"
  }
}
```

#### 4. 渐进式处理策略

**分段处理流程**:
1. **任务分解**: 将大型任务分解为多个小任务
2. **优先级排序**: 根据重要性和依赖关系排序
3. **批次处理**: 按批次处理，每批次保持上下文连贯
4. **结果整合**: 将分段结果整合为完整输出
5. **质量检查**: 对整合结果进行一致性检查

**示例 - 长章节创作**:
```
1. 分解为多个场景段落
2. 为每个段落提供必要上下文
3. 逐段生成内容
4. 段落间进行连贯性检查
5. 整合为完整章节
6. 全章节质量评估
```

### 四、Agent-AI助手解析目标小说并导入架构的工作流程

#### 1. 小说文本预处理

#### 2. 智能信息提取
**人物信息提取**:
- 识别所有出现的人物角色

**世界观设定提取**:

- 提取修炼体系和力量设定
- 分析政治结构和社会制度
- 收集文化背景和历史信息

**剧情结构分析**:
- 识别主线和支线剧情
- 分析剧情发展脉络
- 提取关键冲突和转折点
- 构建时间线和事件序列

#### 3. 数据结构化处理
**自动分类存储**:
- 将提取的信息按类型分类
- 自动填充到对应的数据管理页面
- 建立数据间的关联关系
- 生成初始的项目架构

**一致性验证**:
- 检查提取信息的逻辑一致性
- 识别可能的矛盾和冲突
- 标记需要人工确认的内容
- 生成验证报告

#### 4. 交互式完善
**用户确认流程**:
- 展示提取结果供用户确认

- 提供批量编辑和调整功能
- 支持增量导入和更新

**智能建议**:
- 基于分析结果提供优化建议
- 识别可能遗漏的重要信息
- 建议补充的设定内容
- 提供结构优化方案
